7.0 Fix Contents

In addition to including resolutions to product defects, a fix may also include 
enhancements to existing functionality and features. Any of these items can 
result in new or changed built-in services,APIs, or configuration parameters. 
Review the fix contents carefully before installing the fix.

This fix is cumulative, which means that it includes all previous fixes for the 
release.

PIEAR-1949
When CSRF guard is enabled, the user is not able to enable or disable connections
from the Integration Server Administrator screen if opened using chrome or edge
browser.

This issue is now resolved. 

PIEAR-1920
Suspended connection state is not visible in the Integration Server Administrator,
when the CSRF guard is enabled.

This issue is now resolved. 

PIEAR-1899
In Microservices Runtime, the JDBC polling notifications are not enabled although
the notificationEnabled property is set to true in the application.properties file.

This issue is now resolved. 

PIEAR-1889
The adapter asset properties passed as environment variables arguments in a
docker run command are not substituted for the externalized adapter asset
properties defined in the application.properties file.

This issue is now resolved. 

PIEAR-1888
The listeners in webMethods Adapter for Websphere MQ, which are in pending enabled
state fail to change to enabled state when Integration Server exits the quiesce
mode.

This issue is now resolved. 

PIEAR-1733
When the adapter listeners package is loaded before the adapter package, the
listeners are in Pending Enabled state and fail to change to the Enabled state.

This issue is now resolved.

If the adapter listeners package is loaded before the adapter package, then the
listeners are in the Disabled state and you must enable the listeners manually.
You must ensure proper package dependency is maintained to avoid the error. 

PIEAR-1796 (WAR_10.11_Fix1)
In webMethods Adapter for Runtime, when a user generates configuration variable
template using Integration Server Administrator's Microservices > Configuration
variables > Generate Configuration Variables Template, then the configuration
properties for connection passwords are not masked.

This issue is now resolved. 

PIEAR-1790 (WAR_10.11_Fix1)
In webMethods Adapter for Runtime, when a user generates configuration variable
template using Integration Server Administrator's Microservices > Configuration
variables > Generate Configuration Variables Template, then the configuration
properties for SAP listeners are not generated.

This issue is now resolved. 

PIEAR-1788 (WAR_10.11_Fix1)
The webMethods AdapterRuntime (ART) based adapters configuration properties
which can be configured from Integration Server Administrator are available in
configuration variables template generated by Microservices Runtime.
For more information, see the adapter installation and user guide. 

PIEAR-1673 (WAR_10.11_Fix1)
When parallel asset initialization is enabled for an adapter, reloading a package
with listener notification assets is causing a deadlock.
 
This issue is now resolved. 

Enhancements for WMIO

PIEAR-1919
In webMethods Adapter for SAP, the httPassword field in routing notification is
not considered as a password field. This is applicable to SAP® ERP in
webMethods.io Integration only. 

PIEAR-1787 (WAR_10.11_Fix1)
Added APIs to support SAP® ERP listeners and listener notifications in
webmethods.io. 

PIEAR-1623 (WAR_10.11_Fix1)
New APIs are added to get Adapter's metadata and to call multiple resource
domain calls in a single API for the Database Application in webmethods.io.

10.0 Globalization

suite and includes support for operation in any country, locale, or language as
